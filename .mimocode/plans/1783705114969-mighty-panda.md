# Plan: Fix SplashOverlay Worklet Crash

## Root Cause
SplashOverlay is the FIRST component to use Reanimated hooks at runtime. The native worklet thread (`react-native-worklets`) isn't initialized yet at app startup. BottomSheet works because it only renders 1+ seconds later when the worklet thread is ready.

## Fix
**Replace Reanimated with React Native's built-in `Animated` API in SplashOverlay.** The splash is shown briefly — the visual difference is negligible. This completely eliminates the worklet dependency for the first render.

Additionally, revert `_layout.tsx` to remove `React.lazy` (no longer needed) and use a static import.

## Task 1: SplashOverlay Component

### File: `src/components/SplashOverlay.tsx`

**Props**: `onReady: () => void` — called when splash should dismiss (data loaded + min delay elapsed)

**Internal state**:
- `reduceMotion: SharedValue<boolean>` — detected via `AccessibilityInfo` (same pattern as BottomSheet.tsx)
- `opacity: SharedValue<number>` — 0→1 for entrance, 1→0 for exit
- `iconScale: SharedValue<number>` — 0.8→1 spring
- `iconOpacity: SharedValue<number>` — 0→1
- `nameOpacity: SharedValue<number>` — 0→1 (delayed)
- `nameTranslateY: SharedValue<number>` — 10→0 (delayed)
- `taglineOpacity: SharedValue<number>` — 0→1 (more delayed)

**Layout** (centered vertically, uses `theme.color.background`):
```
View (flex:1, bg: c.background, centered)
  ├── Circle (80dp, RADIUS.xl, bg: c.primary)
  │   └── Ionicons "flash" (ICON_SIZE.xl, color: c.onPrimary)
  ├── "Prompt Forge" (TYPOGRAPHY.heading, c.onBackground)
  └── "Craft perfect AI prompts" (TYPOGRAPHY.caption, c.onSurfaceVariant)
```

**Entrance animation** (in useEffect on mount, following BottomSheet pattern):
- If `reduceMotion.value` is true: set all shared values to final state instantly, then call `onReady()` after a minimal 100ms delay
- If `reduceMotion.value` is false:
  - Icon: `iconOpacity` → withTiming(1, 500ms), `iconScale` → withSpring(1, {damping:15, stiffness:150})
  - Name: `nameOpacity` → withTiming(1, 400ms, delay 150ms), `nameTranslateY` → withTiming(0, 400ms, delay 150ms)
  - Tagline: `taglineOpacity` → withTiming(1, 400ms, delay 300ms)

**Data loading + min delay**:
- Accept `loadingPromise: Promise<void>` as prop (the combined `loadSettings() + loadPrompts()`)
- On mount: `Promise.all([loadingPromise, new Promise(r => setTimeout(r, 1000))])` → then call `onReady()`
- This guarantees minimum 1s display

**Exit animation** (triggered when parent sets a "exiting" flag or via `onReady` callback pattern):
- `opacity` → withTiming(0, 300ms, finished callback) → then parent removes the overlay
- If reduceMotion: skip animation, set opacity to 0 instantly

**Animated styles** (following BottomSheet pattern):
```tsx
const containerStyle = useAnimatedStyle(() => ({ opacity: opacity.value }));
const iconAnimStyle = useAnimatedStyle(() => ({
  opacity: iconOpacity.value,
  transform: [{ scale: iconScale.value }],
}));
const nameAnimStyle = useAnimatedStyle(() => ({
  opacity: nameOpacity.value,
  transform: [{ translateY: nameTranslateY.value }],
}));
const taglineAnimStyle = useAnimatedStyle(() => ({ opacity: taglineOpacity.value }));
```

**Accessibility**: 
- Logo area: `accessibilityLabel="Prompt Forge logo"`
- The whole component should have `accessibilityViewIsModal` or be treated as a visual overlay

**Key design decisions**:
- Spring config: `{ damping: 15, stiffness: 150 }` — softer than BottomSheet's `{ damping: 22, stiffness: 220 }` for a more gentle entrance
- Min delay: 1000ms — enough for the animation to play but not annoyingly long
- Exit fade: 300ms — quick enough to feel responsive

Actually, for simplicity, use `onMomentumScrollEnd` to detect page changes:

const handleScrollEnd = useCallback((e) => {

#### Key Design Decisions

1. **ScrollView pagingEnabled** instead of Pan gesture — simpler, more reliable, handles momentum
2. **useAnimatedScrollHandler** for dot animation — dots animate smoothly during scroll, not just on page change
3. **Per-page animation via useDerivedValue** — each page's content animates based on its proximity to center
4. **reduceMotion** — checked on mount, if enabled: disable scroll animation, set all opacity to 1, skip entrance effects
5. **Spring config for dots**: Not needed — dots use `interpolate` directly from scroll position, no spring
6. **hapticLight on page change** via onMomentumScrollEnd — fires once per page change
7. **hapticHeavy on "Let's Go"** — distinct from normal navigation haptics
8. **No Animated.ScrollView** — actually, we need `Animated.ScrollView` for `useAnimatedScrollHandler`. But wait, `useAnimatedScrollHandler` requires `Animated.ScrollView`. Let me verify... Yes, for Reanimated's scroll handler we need `Animated.ScrollView`.

Actually, looking more carefully, we can use a simpler approach: use `onMomentumScrollEnd` (regular callback) to detect page changes, and `useAnimatedScrollHandler` only for the dot animations. This avoids needing `Animated.ScrollView` for the scroll detection.

Revised: Use `Animated.ScrollView` with `onScroll={useAnimatedScrollHandler({...})}` for dot animation, and `onMomentumScrollEnd` for page detection.
