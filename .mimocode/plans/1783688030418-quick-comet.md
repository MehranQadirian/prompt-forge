# Three UI Fixes

## 1. Animated container swap

**Problem**: Conditional rendering (`!selectionMode ? A : B`) causes instant swap — React unmounts one and mounts the other with no transition.

**Fix**: Render BOTH always, control visibility via animation opacity/translateY. The `pointerEvents` prop blocks interaction on the hidden one.

### index.tsx

Change from conditional rendering to always rendering both:

```tsx
<View style={styles.header}>
  <Animated.View style={headerAnimStyle} pointerEvents={selectionMode ? 'none' : 'auto'}>
    <SearchBar ... />
  </Animated.View>
  <Animated.View style={bulkAnimStyle} pointerEvents={selectionMode ? 'auto' : 'none'}>
    ...bulk toolbar...
  </Animated.View>
</View>
```

The `headerAnimStyle` and `bulkAnimStyle` already animate opacity and translateY. Just need to render both instead of conditionally.

---

## 2. Reset selection mode on tab switch

**Problem**: Selection mode persists when switching tabs.

**Fix**: Add `setSelectionMode(false)` and `setSelectedIds(new Set())` to the `useFocusEffect` cleanup.

### index.tsx

In the existing `useFocusEffect` cleanup:

```tsx
useFocusEffect(
  useCallback(() => {
    return () => {
      setSearchVisible(false);
      setSearchQuery('');
      setFilterCategory(null);
      setShowNewCategory(false);
      setSelectionMode(false);      // ADD
      setSelectedIds(new Set());    // ADD
    };
  }, [...])
);
```

## 3. Color palette visibility on light themes

**Problem**: Some `PROMPT_COLORS` (yellow `#F1FA8C`, cyan `#8BE9FD`) have poor contrast on light backgrounds.

**Fix**: Add a 1dp border to each color circle using `c.outline` (visible border token). This ensures every color is distinguishable regardless of theme.

### ColorGridSheet.tsx

Add border to `colorCircle`:

```tsx
borderColor: isSelected ? c.onBackground : c.outline,
borderWidth: 1,
```

Always show a 1dp border (not just when selected). The `c.outline` token is designed for visible borders on all themes.
