# Swipe Card Fixes — Round 2

## Root Cause Analysis

### Issue 1: Difficult swiping (4 gesture modes missing)
**Root cause**: `onUpdate` uses raw `e.translationX` (distance from gesture start) instead of `startX + e.translationX`. When card is already open at `-72` and user drags right `+50`, the card jumps to `+50` instead of `-22`. The card position resets to the gesture start, not continuing from its current position.

**Also**: `failOffsetY([-8, 8])` is too tight. In a FlatList, even 1-2px of vertical drift during a horizontal swipe kills the pan gesture, making it feel unreliable.

**Fix**: Track `startX` in `onStart`, use `startX + e.translationX` in `onUpdate`, increase `VERTICAL_LOCK` to 20.

### Issue 2/3: Border radius not animating
**Root cause**: Card and action buttons use fixed `borderRadius: RADIUS.lg` (16). No animated per-corner radius.

**Fix**: Derive per-corner radius from progress values. When swiping right (left action revealed), left corners become 0, right stay 16. Vice versa. Same for action buttons — the side facing the card gets 0 radius.

### Issue 4: Delete requires two taps
**Root cause**: `Gesture.Exclusive(panGesture, tapGesture)` — the tap gesture fires on the FIRST tap to close the card, consuming the touch event before the action button's `onPress` fires. The button only receives the press on the second tap when the card is already closed.

**Fix**: Remove the tap gesture entirely. Instead, close the card via a `Pressable` overlay that covers the card when open, or handle tap-to-close in the action button's onPress itself (close card → execute action).

**Final approach for Issue 4**:

The real fix is to NOT use `Gesture.Exclusive` with a tap gesture. Instead:

1. Use only `Gesture.Pan()` 
2. When the card is open and user taps (no pan movement), the pan gesture will fire `onStart` → `onEnd` with `translationX ≈ 0`. In `onEnd`, if `translationX` is near 0 and velocity is near 0, treat it as a tap and close.
3. This way, the first tap closes the card, and the action buttons receive their own press events directly.

```ts
.onEnd((e) => {
  'worklet';
  isDragging.value = false;
  const velocity = e.velocityX;
  const translation = translateX.value;
  
  // Tap detection: minimal movement and velocity
  const isTap = Math.abs(translation - startX.value) < 5 && Math.abs(velocity) < 100;
  
  if (isTap && translation !== 0) {
    // Tap on open card — close it
    translateX.value = withSpring(0, SPRING_CONFIG);
    runOnJS(clearOpenCard)(cardId);
    return;
  }
  
  // ... existing snap logic
});
```

This way:
- Tap on open card → closes it (first tap works)
- Tap on action button → button's onPress fires directly (no gesture interference)
- Swipe → gesture works normally

### 2. `src/components/cards/SwipeActionRow.tsx` — Border radius fix

The action button should have:
- When on the LEFT side: no border radius on the RIGHT side (topRight, bottomRight = 0)
- When on the RIGHT side: no border radius on the LEFT side (topLeft, bottomLeft = 0)

Pass `progress` as a prop and animate:

```ts
interface SwipeActionRowProps {
  action: SwipeAction;
  side: 'left' | 'right';
  animatedStyle: any;
  progress: any; // SharedValue<number>
  onPress: () => void;
}
```

In the component:
```ts
const buttonStyle = useAnimatedStyle(() => {
  const p = progress.value;
  if (side === 'left') {
    // Left side: left corners = RADIUS.lg, right corners = interpolate(p, [0,1], [RADIUS.lg, 0])
    return {
      borderTopRightRadius: interpolate(p, [0, 1], [RADIUS.lg, 0], Extrapolation.CLAMP),
      borderBottomRightRadius: interpolate(p, [0, 1], [RADIUS.lg, 0], Extrapolation.CLAMP),
    };
  } else {
    // Right side: right corners = RADIUS.lg, left corners = interpolate(p, [0,1], [RADIUS.lg, 0])
    return {
      borderTopLeftRadius: interpolate(p, [0, 1], [RADIUS.lg, 0], Extrapolation.CLAMP),
      borderBottomLeftRadius: interpolate(p, [0, 1], [RADIUS.lg, 0], Extrapolation.CLAMP),

```

Remove the fixed `borderRadius: RADIUS.lg` from the container style. Instead, set all four corners to `RADIUS.lg` as defaults and let the animated style override the relevant ones.
